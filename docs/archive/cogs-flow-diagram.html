🔍 Flow Audit — What's Working, What Needs Attention

✅ Revenue Calculation
FIXED serviceCharges[] correctly added to gross before discount in GlassCoQuotationPrint.tsx. netAmount = subTotal + aptCharges + serviceCharges − discount.
✅ COGS GL — 3 Separate Lines
CODED postDeliveryCOGS() correctly splits into Raw Glass (5111), Cutting Labour (51311), Processing Labour (51312) in one SA transaction.
✅ Service Worker Tracking
CODED ServiceFloorView.tsx captures worker per service. ServiceLogEntry stored in piece.serviceLog[]. Role-filtered dropdown: Cutter, Polish Operator, etc.
✅ Factory Overhead Allocation
CODED 4-company GL posted. Factory: Dr Recv / Cr Cost Allocated. Glassco: Dr 51415-51418 + Dr 52911 / Cr 21141 Due to Factory. Reversible.
✅ Month-End OH Absorption
CODED IAS 2.13 compliant. Rate = (own consumables + Factory shared) ÷ sqft produced. Variance tracked. Dr proportional / Cr 5142.
✅ Payroll GL
CODED hrService.savePayroll() posts Dr 51311/51312/52111 / Cr 21311 on payment. IAS 19 compliant. Idempotency guard on GL-PAY-{co}-{month}.
⚠ Inventory Stock Deduction
PARTIAL MAP updated on store items, but quantity (unrestricted qty) not decremented when glass is cut. Cutting GL debits WIP but inventory qty stays same. Physical count will diverge.
⚠ COGS GL Status = Parked
REVIEW postDeliveryCOGS() posts status=Parked. This means COGS does NOT appear in P&L until Finance manually approves. Delivery P&L will show 0 COGS until then. Consider auto-post if no approval workflow.
⚠ Cutting GL Status = Parked
REVIEW postCuttingGL() status=Parked means WIP never moves to balance sheet until Finance approves. Same issue — real-time WIP balance is always 0 in posted ledger.
⚠ serviceLog → Supabase sqft
VERIFY productionService upsert includes sqft + service_log columns. However, the Supabase table schema must have these columns added via migration. Check if migration was run or if it's silently failing.
🔴 WIP → Finished Goods Transfer
MISSING No GL entry for Dr 11515 Finished Goods / Cr 11513 WIP when piece completes all services and is ready for delivery. Inventory never moves from WIP to FG on the balance sheet.
🔴 Tempering Cost in COGS
GAP getPiecePnL() correctly calculates temperingCost from dispatches. But postDeliveryCOGS() does NOT include a separate COGS line for tempering — only raw glass + labour. Tempering cost is in MAP update but not as a separate COGS line.
🔴 AP Payment to Tempering Vendor
MISSING GL-TEMP posts Dr WIP / Cr AP-Tempering. But when vendor is paid, no payment GL exists. AP 22113 grows forever — no Dr AP / Cr Cash when vendor bill settled.
⚠ Piece.orderId matching
FRAGILE postDeliveryCOGS() filter: p.orderId === order.id || p.orderId === order.orderNo. If either field is undefined, pieces are silently skipped and COGS = 0. Should add fallback logging.
